# 大喜利道場｜デザイン方針

> 企画書.md / 仕様書.md を前提とした舞台美術・UI・キャラクターデザインの方向性検討。
> 「決まった時間にみんなで集まるオンライン大喜利ライブ」というコンセプト、「芸人がネタをする舞台／寄席」というモチーフ、幕間演出・笑いエフェクト・表彰演出などの演出要件、ポイント/熟練度メーター/ガチャという経済・コレクション要素を踏まえて、design-dept視点でトーン・配色・タイポグラフィ・主要画面・キャラクター方向性・段位表現を整理する。

---

## 0. 前提の整理（このアプリのデザイン上の特徴）

- ユーザー体験の芯は「開演の高揚感 → 舞台と客席の入れ替わり → その場の笑い → 表彰の余韻」という**時間軸のあるライブ体験**。静的な画面美しさよりも、**演出（トランジション・エフェクト）の質がUXの核**になる。
- 実装はNext.js + Framer Motion前提なので、リッチなモーション表現を作り込む余地がある。むしろ「動きで語る」設計にした方が本アプリらしさが出る。
- 姉妹アプリ「大喜利育成」が暖色ポップ・丸ゴシック・グミボタンの「スマホゲームらしいソシャゲトーン」を既に確立している。道場は同じトーンをなぞる必要はなく、**「寄席・ライブハウスの熱」を軸にした、少し大人びた/かっこいいゲーム性**で差別化するのが妥当。
- 「かなりゲーム寄りに作りたい」という要望があるため、和風に振り切りすぎて渋くなりすぎないよう、**ゲームUIらしい派手な光・スコア演出・コレクション性**は必ず残す。

---

## 1. 世界観・トーンの方向性

### 選択肢

**A. 和風寄席・伝統演芸路線**
提灯、金縁の緞帳、和柄（青海波・麻の葉）、寄席文字（勘亭流）を全面に出す。落語会のような格式・粋さが出るが、彩度を上げないとゲームとしての派手さが不足しやすく、実装（アニメーション素材）も専用に作り込む必要が出やすい。

**B. 現代的お笑いライブハウス／劇場路線**
暗めの劇場空間、スポットライト、ネオン、拍手・歓声のエネルギー感。都会的でスタイリッシュ、eスポーツ配信番組のような「観る快感」と相性が良く、ゲームらしい派手な光エフェクトとも馴染みやすい。ただし「寄席」らしさは薄まる。

**C. エンタメショー／バラエティ番組路線**
TV収録セットのような電飾・ゴールド×レッドの派手な演出、スコアボード演出、煽りナレーション的なUI。最も“ゲーム的な派手さ”は出しやすいが、企画のコアである「一回性のライブの一体感」「寄席的な間（ま）」よりも賑やかしが勝ってしまうリスクがある。

### 推奨：B寄りのハイブリッド（「ネオ寄席」）

**現代的なライブハウス／劇場の暗さと光を基調にしつつ、緞帳の赤・金の縁取り・提灯的な丸みなど「寄席の記号」をワンポイントで残す**方向を推奨する。

- 理由1：暗転→スポットライト→歓声という演出フローは、企画の核である「幕間の高揚感」「笑いエフェクト」「表彰演出」と最も相性が良く、Framer Motionでの光・パーティクル表現がそのままゲームらしい派手さになる。
- 理由2：和風全振り（A案）だと、若年層に刺さる“ゲーム的な派手さ”との両立が難しく、実装コストも専用アセット（和柄パターン、筆文字）が増える。ワンポイント使用に留めれば、和のモチーフの意味（芸人・舞台・演目）を保ったまま実装負荷を抑えられる。
- 理由3：C案（バラエティ番組）は表彰演出やガチャ演出には合うが、企画書が大事にする「その日限りの一撃」「寄席的な間」という**静と動の緩急**が失われやすい。B案なら、通常時は暗く落ち着いた劇場、盛り上がる瞬間だけ強い光、というメリハリを作れる。

**簡易版の代替案**：もし専用アセット制作コストを絞りたい場合、和柄・筆文字などA案要素は最小限（ロゴ・幕間タイトルのみ）に留め、UIパーツ自体は角丸長方形＋単色のシンプルなゲームUI（B寄り）に寄せてもトーンは成立する。

## 2. カラーパレット

舞台・緞帳・照明・暗転をイメージした配色。**「暗い舞台＋暖色スポットライト」を基調とし、金を装飾・段位の格に、赤を緞帳/アクセントに使う**構成。

| セカンダリ背景（客席画面等の面） | 幕裏紺 | `#1F1D2B` | カード等を置く一段明るい面 |

| ゴールド（段位・表彰・縁取り） | 幕金 | `#E8B84C` | 段位バッジ、枠線、1位演出 |
| ゴールド明 | 光箔 | `#F6D77C` | ハイライト、グラデーション上端 |
| スポットライト（暖色グラデーション） | 灯り橙 | `#FF8A3D` → `#FFD98E` | 舞台照明、注目時のグロー |

| テキスト（サブ） | 灰紫 | `#B7B2C4` | 補足情報、タイムスタンプ等 |

**コントラスト配慮**：背景が暗色主体になるため、本文テキストは`#F5F1E6`系、ボタン内文字は暗色背景上に置く金/白文字を基本とし、WCAG AAのコントラスト比（4.5:1以上）を各主要テキストで確認しながら実装する。緞帳赤(`#C0263F`)の上に白文字を乗せる場合はコントラストが不足しがちなので、太字化や文字サイズ拡大、あるいは背景をより暗い`#8E1A2E`にするなどの調整を推奨。

**姉妹アプリとの差別化**：育成アプリの暖色パレット（明るいオレンジ〜ピンクのポップ系）に対し、道場は**「暗い舞台に暖色の光が差す」明暗のコントラストが強い配色**にすることで、同じ暖色系でも印象を大きく変えられる。

### 2.5 配色改訂（2026-07-13、和風トーンへ調整）
実機を見たユーザーから「ネオ寄席の雰囲気も少し残しつつ、もっと和風な薄茶・緑・畳のイメージにしたい」との要望あり。ベースをダーク系から**畳・木目を思わせる落ち着いた和風トーン**へ寄せ、ネオ寄席由来の金・赤はワンポイントのアクセントとして残す。**この配色はユーザー自身が後で作り込み直す前提の暫定調整**でよい（細部を詰めすぎない）。

| 用途 | カラー名 | HEX | 備考 |
|---|---|---|---|
| ベース背景 | 畳生成り | `#E8DCC4` | 舞台闇に代わるメインの明るい和紙・畳色ベース |
| セカンダリ背景（面・カード） | 薄茶 | `#D8C6A3` | カード等の一段濃い面 |
| 深み・縁取り | 焦げ茶 | `#5B4530` | 縁線・濃色テキスト・木枠のイメージ |
| アクセント緑（畳縁・成功表示） | 畳縁緑 | `#5C7A54` | 畳の縁、成功/OK表示、落ち着いた差し色 |
| ネオ寄席の名残（アクセントのみ） | 緞帳赤・幕金 | `#C0263F` / `#E8B84C` | CTA・段位・表彰演出など「ここぞ」の場面に限定して残す |
| テキスト（メイン） | 墨色 | `#2E2A24` | 明るい背景での本文用濃色テキスト |

- 暗転・幕間演出やライブ本番中の一部演出（笑いエフェクト等）は、従来の舞台闇＋暖色スポットの世界観を残してよい（すべてを和風の明るいトーンに統一する必要はない）。
- 実装時は「ベースは和風の明るい配色、ここぞという演出・CTA・段位表現にネオ寄席の金/赤を差す」という役割分担を意識する。

## 3. タイポグラフィ

**「和の芸能感」はワンポイントの見出し/ロゴに限定し、本文は現代的な可読性を優先する**方針。

| 用途 | フォント方向性 | 備考 |
|---|---|---|
| アプリロゴ・幕間タイトル（「大喜利道場」表示など） | 筆文字系・寄席文字風（例：Yuji Syuku / Yuji Boku、または勘亭流風の商用和文フォント） | 使用箇所を絞り込み、ロゴ・幕間演出・表彰演出の大見出しのみに使用。多用すると可読性が落ちるため本文には使わない |
| 画面見出し・強調テキスト | 太めの角ゴシック（例：Zen Kaku Gothic New Black/Bold、Noto Sans JP Black） | ゲームUIらしい力強さ。姉妹アプリの丸ゴシック（Zen Maru Gothic）と差別化し「大人っぽいライブ感」を出す |
| 本文・UIラベル | Noto Sans JP / Zen Kaku Gothic New（Regular〜Medium） | 可読性最優先、長文の回答テキスト表示にも耐える |
| タイマー・スコアなどの数値表示 | 太めの角ゴシック＋等幅数字（tabular-nums指定） | デジタル感を出しつつ視認性を確保。凝った専用デジタルフォントは可読性リスクがあるため、既存ゴシックの太字＋字間調整で十分 |

タイポグラフィの役割分担を明確にすることで、「和の芸能感」と「現代的な読みやすさ」を**混在させず使い分ける**設計にする。

## 4. 主要画面のビジュアルコンセプト

### 4.1 幕間演出（暗転→タイトル表示）

- 暗転：画面全体が舞台闇（`#12101A`）へ0.4〜0.6秒でフェード。BGM/効果音がある場合は暗転と同時に「間」を作る無音〜小さな太鼓的効果音。
- 中央にスポットライトが絞り込まれるようなラジアルグラデーション（`#FF8A3D`→透明）が浮かび上がり、その中に金縁の筆文字ロゴ「大喜利道場」がフェード＋わずかな拡大でせり上がる。
- 開演時のフル幕間と、組切替時のミニ幕間を分ける：
  - **フル幕間（開演前）**：全画面暗転＋タイトル演出＋「第◯回」等のサブテキスト、1.5〜2秒程度。
  - **ミニ幕間（組切替）**：全画面までは暗転せず、上下に緞帳赤の帯が閉じるようなワイプ演出＋「◯組目 登場」のテキストが横スライドで入る、0.5〜0.8秒程度。テンポを落とさないよう短く。

### 4.2 ライブ観戦（客席/審査員）画面（改訂：画面固定・使い捨て表示、2026-07-13確定）

- **画面全体はビューポート内に完全固定**（スクロール発生禁止）。過去の回答を溜めるタイムライン/一覧表示は廃止。
- 上部固定エリア：現在のお題。**審査サイクル外は控えめな一行表示**、**審査サイクル中はカードが拡大してスポットライトのグロー付きで画面中央上部に強調表示**され、残り10秒を円形プログレスリング（灯り橙→緞帳赤へ色が変化するカウントダウン）で示す。
- 中央：**「今まさに審査されている1件」の回答だけを大きく中央表示する使い捨てカード**。カードは幕裏紺の半透明パネル＋金の細枠、演者名＋回答テキスト、拡大・発光でスポットライトを浴びているように見せる。採点確定と同時に短い演出（例：フェード＋縮小しながら消える）で画面から消去し、次の回答が来るまでは空の舞台（またはシンプルな待機表示）にする。
- 下部固定エリア：
  - 審査サイクル中：0/1/2の採点ボタンを大きな円形ボタンで表示（採点0＝グレー、1＝灯り黄、2＝緞帳赤、押下時に光が広がるフィードバック）。誤操作防止のため他の操作ボタンはこの間非活性化しトーンダウン。
  - サイクル外：拍手ボタン（連打可、押すたびに小さな手のひらアイコンが舞い上がる）とテンプレツッコミボタン（ピル型、金縁）を横並びで配置。育成アプリの「グミボタン」と似た形状を採用しても良いが、色は暖色ポップではなく金・赤系の舞台トーンにする。
- レイアウト：モバイルは縦一列（お題→使い捨て回答カード→採点/演出ボタン）。デスクトップは2カラム（左：使い捨て回答カード表示エリア、右：採点/演出パネルを固定表示）とし、視線移動を減らす。いずれの場合も画面の高さに収まるよう各要素のサイズを調整する（`100dvh`基準を推奨、モバイルブラウザのアドレスバー分の高さ変動に対応するため）。

### 4.3 回答入力（舞台）画面

- 主役はお題テキスト。画面中央にスポットライトの光の輪の中に浮かぶように大きく表示し、「今、舞台に立っている」感を演出する。
- 残り持ち時間：上部に劇場の時計を思わせるカウントダウン表示。通常は灯り橙、残り10秒を切ると緞帳赤に変わり点滅させ緊張感を出す。審査サイクル中は一時停止していることが視覚的に分かるよう、カウントダウンの周囲を暗くグレーアウトする。
- 回答入力欄：画面下部に大きめのテキストエリア。残り回答回数はチップ（●●○のような3つのドット、送信済みは金で塗りつぶし）で表示。
- 送信キュー状態（「他の回答を審査中」）：自分のスポットライトが一瞬外れるような減光演出＋控えめなテキスト表示で、待機中であることを視覚的に伝える。
- 採点確定時：自分の回答が獲得したスコアを、紙吹雪の演出とともに大きくポップアップ表示（例：「+2」の数字がバウンドしながら出現）。

### 4.4 笑いエフェクト・拍手エフェクトの視覚表現

- **笑いエフェクト（過半数2点）**：画面全体が一瞬明るくフラッシュ（灯り橙〜光箔色）、金の紙吹雪と光の粒子が舞台上から降り注ぐ。複数のスポットライトが交差するように光の筋が増える演出とあわせ、擬音風のテキストスタンプ（「ドッ」「ウケた」等）が弾けて消える。0.8〜1.2秒程度で収束させ、テンポを崩さない。
- **拍手エフェクト**：客席側から小さな手のひらアイコンが画面下部から複数舞い上がる。連打頻度に応じてパーティクル密度が増える設計にすると、盛り上がりの熱量が可視化されて良い。
- **テンプレツッコミ**：吹き出し風のスタンプが画面を横切ってスライドイン→消える。フォントはロゴと共通の筆文字/寄席文字風にすると、この一瞬だけ和のテイストが効いて「合いの手」らしさが出る。

### 4.5 表彰演出（3位→2位→1位）

- 全体を再度暗転させ、緞帳が開くようなワイプで開始。3位から1人ずつスポットライトが当たり、名前・段位・スコアが金縁の額縁カードでせり上がる形で発表。
- 1位発表時が最大の演出：紙吹雪・光の粒子・金のテープ演出をこのタイミングだけ最大化し、他の順位発表よりワンランク派手にすることでカタルシスを作る。
- 「本日のベストアンサー」は、幕間演出と同じ「緞帳が開く」モチーフを再利用し、回答文そのものを大写しにする。演出の一貫性（開幕＝始まりの緞帳、ベストアンサー＝一撃の緞帳）を持たせることで世界観の統一感を出す。

## 5. キャラクター/アバターのデザイン方向性

### 選択肢

**A. 芸人風の人型キャラ**
着物・羽織・スーツなど「舞台に立つ演者」をモチーフにしたデフォルメ人型キャラ。衣装の着せ替え・小物（扇子、マイク、めがね等）の装備という収集要素と直結しやすい。

**B. 動物擬人化キャラ**
姉妹アプリ「大喜利育成」がすでに動物擬人化・育成ゲーム路線を採用しているため、道場で同じ方向を採ると世界観が被り、ユーザーから見て2つのアプリの違いが分かりにくくなるリスクがある。

**C. シンプルなアイコン風（顔なしオブジェクトモチーフ）**
提灯・座布団・マスクなど、演芸に関わるモノをモチーフにした記号的アバター。実装・量産コストは最も低いが、「自分の分身」としての愛着形成やコレクション欲求はA・Bに比べて弱くなりやすい。

### 推奨：A案（芸人風の人型キャラ、SDデフォルメ）

- 理由1：企画の核は「参加者は演者、全員がいつか舞台に立つ」こと。**人型の“演者アバター”は企画テーマと直結**し、舞台に立つ瞬間の演出（スポットライト、表彰）ともキャラクターとして自然に馴染む。
- 理由2：衣装（着物・スーツ・パンク衣装など）・アイコンパーツ（扇子・マイク・めがね等の小物）というガチャの収集軸は、**人型キャラの方が「着せ替え」として直感的に伝わりやすく**、レアリティ別の見た目差も作りやすい（例：SSRは豪華な特殊衣装、Nは普段着）。
- 理由3：動物擬人化は姉妹アプリ「大喜利育成」と方向性が重複し、ユーザーが2つのアプリを混同するリスクがある。人型に振ることで**「育成＝動物を育てるソロゲーム」「道場＝人間としてライブに立つ」という住み分け**が視覚的にも明確になる。
- 頭身・タッチ：フルイラストのリアル頭身ではなく、**2〜3頭身程度のSD（スーパーデフォルメ）キャラ**を推奨。理由は(1)量産・差分制作のコストが抑えられる、(2)ゲームUIの中で小さく表示されるアイコン用途にもデフォルメの方が視認性が高い、(3)舞台演出の派手さとポップさのバランスが取りやすいため。

**簡易版の代替案**：フルボディのSDキャラ制作コストが厳しい場合、「顔＋肩から上の簡易人型アバター」に絞り、衣装は上半身の記号的レイヤー（襟・羽織・帽子など）だけを重ねる簡略版でも、ガチャ収集の面白さと演者感はある程度両立できる。将来的にフルボディへ拡張する前提の暫定案として位置づけると良い。

## 6. 段位（ビギナー→達人）の視覚的な見せ方

- **バッジ**：円形のメダル/紋章風デザイン。段位が上がるごとに素材感を変える（銅→銀→金→錦などの質感グラデーション）ことで、実数を出さずとも「格が上がった」ことが一目でわかるようにする。モチーフには扇子・太鼓・纏（まとい）など寄席・演芸系のアイコンをワンポイントであしらう。
- **称号カード（マイページ）**：寄席の高座に掲げる「めくり」（名前が墨書きされた木札・紙札）を模したカードデザインにし、段位名を筆文字系フォントで表示。段位が上がるほど背景に金の箔押し風パターンが増えていく演出にすると、コレクション性・成長実感が視覚的に積み上がる。
- **ライブ中の名札**：情報量を絞り、段位アイコンのみを小さく表示（実数のメーターやスコアは仕様書の方針どおり非表示）。対戦相手を過度に意識させず、"称号"としての誇らしさだけを見せる。
- **段位間の視覚的ジャンプ**：単純な色替えだけでなく、上位段位（例：名人・達人）では枠のエフェクト（微発光アニメーション等）を追加するなど、視覚的な「格の差」を演出的にも表現すると達成感が強まる。
